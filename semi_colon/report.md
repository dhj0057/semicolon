# 🚀 개발 입문 핵심 개념 정리

## 1. Git vs GitHub: "도구"와 "공간"의 차이
많은 입문자가 혼동하지만, 둘은 **'버전 관리 시스템'**과 이를 호스팅하는 **'서비스'**라는 명확한 차이가 있습니다.

* **Git (도구):** 로컬 컴퓨터에서 파일의 변경 사항을 기록하는 **소프트웨어**입니다. 코드의 과거 상태를 저장(Commit)하고 필요할 때 되돌리는 역할을 합니다.
* **GitHub (플랫폼):** Git으로 관리하는 프로젝트를 클라우드에 올릴 수 있는 **온라인 저장소**입니다. 전 세계 개발자들과 협업하고 코드를 공유하는 커뮤니티 역할을 합니다.

## 2. Markdown(.md)을 사용하는 이유
마크다운은 텍스트 기반의 가벼운 마크업 언어로, 개발자들에게는 **표준 문서 양식**과 같습니다.

### 왜 사용하는가?
1.  **생산성:** 마우스 없이 키보드만으로 문서 구조(제목, 목록, 코드 블록)를 빠르게 잡을 수 있습니다.
2.  **버전 관리:** 일반 텍스트 파일이므로 Git을 통해 문서의 변경 이력을 추적하기 쉽습니다.
3.  **플랫폼 호환성:** GitHub README, Notion, 기술 블로그(Velog, Tistory) 등에서 널리 쓰입니다.

### 주로 쓰이는 곳
* **README.md:** 프로젝트의 설치 방법 및 사용법 설명.
* **Issue/PR:** 협업 시 질문을 하거나 코드 리뷰를 요청할 때.
* **기술 문서:** API 명세서나 라이브러리 가이드라인 작성.

## 3. 프로그래밍 언어의 종류와 주 사용처
언어는 해결하고자 하는 문제에 따라 적절한 도구를 선택하는 것이 중요합니다.

| 언어 | 주 사용처 | 특징 |
| :--- | :--- | :--- |
| **Python** | 데이터 분석, AI, 자동화 스크립트 | 문법이 간결하여 배우기 쉽고 라이브러리가 풍부함. |
| **JavaScript** | 웹 프론트엔드, 서버(Node.js) | 웹 브라우저에서 실행되는 유일한 언어이며 생태계가 매우 큼. |
| **Java** | 백엔드(Spring), 안드로이드 앱 | 대규모 엔터프라이즈 시스템과 국내 공공기관 사업의 표준. |
| **Swift / Kotlin** | iOS 앱 / 안드로이드 앱 | 각 모바일 플랫폼에 최적화된 네이티브 앱 개발 언어. |
| **C / C++** | 게임(Unreal), OS, 하드웨어 제어 | 성능이 매우 뛰어나지만 메모리 관리 등 학습 난이도가 높음. |

## 4. 라이브러리, 프레임워크, API의 이해
세 개념은 모두 '남이 만든 코드'를 활용한다는 공통점이 있지만, **제어권**과 **목적**이 다릅니다.

> **💡 한 줄 요약**
> * **라이브러리 (Library):** 내가 필요할 때 부르는 **도구 세트**.
> * **프레임워크 (Framework):** 내가 들어가서 작업해야 하는 **미리 짜여진 틀**.
> * **API (Interface):** 프로그램끼리 소통하기 위한 **데이터 통로/약속**.

### 상세 차이점
1.  **라이브러리:** 코드의 흐름을 **사용자(나)**가 결정합니다. (예: 날짜 계산을 위해 Moment.js 사용)
2.  **프레임워크:** 코드의 흐름을 **프레임워크**가 결정하며, 사용자는 그 규칙에 맞춰 코드를 채워 넣습니다. (예: Django, Spring)
3.  **API:** 내부 구현은 몰라도 되며, 정해진 규칙에 따라 요청을 보내면 결과를 돌려받습니다. (예: 구글 지도 API)
